# 개인정보처리방침

시행일: 2026년 3월 30일

redniche(이하 "운영자")는 Discord 기반 금융·시장 정보 리서치 봇 서비스(이하 "서비스")를 운영함에 있어 「개인정보 보호법」 제30조에 따라 정보주체의 개인정보를 보호하고 이와 관련한 고충을 신속하고 원활하게 처리할 수 있도록 다음과 같이 개인정보처리방침을 수립·공개합니다.

## 1. 개인정보의 처리 목적

운영자는 다음 목적 범위에서만 개인정보를 처리합니다.

1. Discord 봇 명령 실행 및 결과 전달
2. 사용자 요청 식별, 권한 검증, 화이트리스트 정책 적용
3. 리서치 실행, 보고서 생성, 파일 업로드 및 실행 로그 관리
4. 오류 확인, 보안 대응, 오남용 방지 및 서비스 안정성 확보
5. 이용자 문의 대응 및 분쟁 처리

운영자는 위 목적 외 용도로 개인정보를 이용하지 않으며, 이용 목적이 변경되는 경우 관련 법령에 따라 필요한 조치를 이행합니다.

## 2. 처리하는 개인정보 항목

운영자는 서비스 운영 과정에서 다음 개인정보 또는 개인정보에 해당할 수 있는 정보를 처리할 수 있습니다.

1. Discord 식별 정보
   - Discord 사용자 ID
   - Discord 서버 ID
   - Discord 채널 ID 또는 스레드 ID
   - Discord 사용자명, 표시명, 프로필상 식별 정보
2. 서비스 이용 정보
   - 명령 실행 시각
   - 호출한 명령어와 선택한 스킬 이름
   - 질문, 프롬프트, 후속 입력 내용
   - 실행 결과 요약, 상태 메시지, 오류 로그
   - 생성된 보고서 파일 경로, HTML/PNG 산출물 정보
3. 기술 및 보안 정보
   - 시스템 로그
   - 프로세스 실행 메타데이터
   - 비정상 이용 탐지에 필요한 접속 및 사용 기록

운영자는 주민등록번호, 계좌번호, 카드번호, 비밀번호, API 키, 민감한 금융정보 등 불필요한 민감정보의 입력을 요구하지 않습니다. 이용자는 위와 같은 정보를 서비스에 입력하지 않아야 합니다.

또한 이용자가 질문이나 프롬프트에 개인정보를 임의로 포함하는 경우에도, 운영자는 이를 별도의 개인정보 데이터베이스로 사전 분류·정리·식별하여 저장하는 방식을 채택하지 않습니다. 서비스는 원칙적으로 명령 실행에 필요한 최소한의 운영 메타데이터와 결과물 파일만 보관하며, 질문 본문에 포함된 개인정보를 찾기 위한 별도 검색·색인 체계를 운영하지 않습니다.

## 3. 개인정보의 처리 및 보유 기간

운영자는 개인정보를 처리 목적 달성에 필요한 범위 내에서 보유하며, 원칙적으로 처리 목적이 달성되면 지체 없이 파기합니다. 다만 아래와 같이 운영상 필요한 범위에서 일정 기간 보관할 수 있습니다.

1. Discord 명령 실행 로그, 오류 로그, 실행 메타데이터
   - 보유기간: 생성일로부터 최대 1년
   - 보유목적: 장애 분석, 보안 대응, 오남용 방지, 운영 이력 확인
2. 생성된 보고서 파일, JSON 결과, 로컬 실행 산출물
   - 보유기간: 생성일로부터 최대 1년
   - 보유목적: 이용자 요청 재확인, 문제 재현, 서비스 품질 개선
3. 문의 또는 분쟁 대응 기록
   - 보유기간: 처리 완료 후 최대 3년
   - 보유목적: 민원 대응, 법적 분쟁 대비

법령에 따라 별도 보존이 필요한 경우 해당 법령이 정한 기간 동안 보관할 수 있습니다.

## 4. 개인정보의 제3자 제공

운영자는 원칙적으로 정보주체의 개인정보를 제3자에게 제공하지 않습니다.

다만 다음의 경우에는 예외로 할 수 있습니다.

1. 정보주체가 사전에 동의한 경우
2. 법령에 특별한 규정이 있거나 법적 의무 이행을 위하여 필요한 경우
3. 수사기관, 감독기관 또는 법원의 적법한 절차에 따른 요청이 있는 경우

## 5. 개인정보 처리의 위탁 및 외부 서비스 이용

운영자는 서비스 운영을 위하여 다음 외부 서비스를 이용할 수 있습니다. 이 과정에서 개인정보가 해당 서비스의 인프라를 통하여 처리될 수 있습니다.

1. Discord
   - 위탁 또는 이용 업무: 명령 수신, 메시지 송수신, 파일 업로드, 사용자 식별
2. AI model service 또는 관련 모델 제공 인프라
   - 위탁 또는 이용 업무: 프롬프트 처리, 리서치 실행, 결과 생성

운영자는 외부 서비스 이용 시 개인정보가 목적 외로 처리되지 않도록 필요한 범위에서 관리하며, 관련 법령에 따라 계약상 보호조치 또는 관리적 조치를 검토합니다.

## 6. 개인정보의 국외 처리 가능성

서비스는 Discord 및 OpenAI 등 해외 인프라를 포함한 외부 서비스를 이용할 수 있으므로, 이용자가 입력한 정보와 서비스 처리 과정에서 발생한 데이터가 대한민국 외 지역의 서버에서 처리되거나 저장될 수 있습니다.

운영자는 서비스 운영에 필수적인 범위 내에서만 이러한 외부 서비스를 이용하며, 이용자에게 불필요한 민감정보 입력을 요구하지 않습니다. 이용자는 개인정보, 인증정보, 비공개 금융정보의 입력을 자제하여야 합니다.

## 7. 개인정보의 파기절차 및 파기방법

운영자는 개인정보 보유기간의 경과, 처리목적 달성 등 개인정보가 불필요하게 되었을 때에는 지체 없이 해당 개인정보를 파기합니다.

1. 파기절차
   - 파기 사유가 발생한 개인정보를 식별한 후 지체 없이 삭제 또는 파기합니다.
   - 법령에 따라 별도 보존이 필요한 정보는 일반 데이터와 분리하여 보관한 뒤 해당 보존기간 종료 후 파기합니다.
2. 파기방법
   - 전자적 파일 형태의 정보는 복구 또는 재생이 어려운 기술적 방법으로 삭제합니다.
   - 종이 문서 형태의 정보는 분쇄 또는 소각 등 복원이 불가능한 방법으로 파기합니다.

## 8. 정보주체 권리행사에 관한 안내

운영자는 서비스 운영상 필요한 최소 메타데이터만 제한적으로 보관하며, 이용자가 질문·프롬프트에 임의로 입력한 개인정보를 별도로 식별·색인·분류하여 관리하지 않습니다. 따라서 일반적인 형태의 개인정보 열람, 정정, 삭제, 처리정지 요구에 대하여 운영자가 실질적으로 특정 데이터를 식별하거나 검색하여 응답할 수 없는 경우가 있을 수 있습니다.

운영자는 이러한 권리행사 요청을 처리하는 과정에서도 이용자에게 추가적인 개인정보, 신분증 사본, 금융정보 또는 기타 과도한 확인자료의 제출을 요구하지 않습니다.

다만 운영자가 이미 보유 중인 최소한의 정보만으로 요청 대상을 확인할 수 있고, 관련 법령상 처리의무가 인정되는 경우에는 그 범위에서 검토할 수 있습니다. 반대로 아래와 같은 경우에는 요청을 처리하지 않거나 제한할 수 있습니다.

1. 요청 대상 개인정보를 운영자가 별도로 보유하거나 식별할 수 없는 경우
2. 요청 처리를 위해 정보주체에게 추가 개인정보 제출을 요구하게 되는 경우
3. 법령에서 열람, 정정, 삭제 또는 처리정지를 제한하는 경우
4. 다른 사람의 권리 또는 서비스 보안을 침해할 우려가 있는 경우

이용자는 서비스에 개인정보를 입력하지 않는 방식으로 자신의 개인정보를 가장 효과적으로 보호할 수 있습니다.
