# 개인정보처리방침

시행일: 2026년 3월 30일

redniche(이하 "운영자")는 1인 개발자로서 Discord 기반 금융 리서치 봇 서비스(이하 "서비스")를 운영합니다. 운영자는 「개인정보 보호법」 제30조에 따라, 서비스 운영 과정에서 처리하는 개인정보와 그 목적을 아래와 같이 안내합니다.

## 1. 처리 목적

운영자는 다음 목적 범위에서만 개인정보를 처리합니다.

1. Discord 봇 명령 실행 및 결과 전달
2. 사용자 식별, 권한 확인, 오남용 방지
3. 보고서 생성, 파일 산출물 관리, 오류 확인
4. 구독 상태 확인, 결제 반영, 영수증 확인
5. 환불, 취소, 분쟁, 문의 대응

## 2. 처리하는 개인정보 항목

운영자는 서비스 운영에 필요한 최소한의 정보만 처리합니다.

1. Discord 정보
   - Discord 사용자 ID
   - Discord 서버 ID
   - Discord 채널 또는 스레드 식별자
   - 사용자명 또는 표시명
2. 서비스 이용 정보
   - 실행한 명령어
   - 실행 시각
   - 오류 로그 및 상태 정보
   - 생성된 보고서 및 산출물 경로
3. 결제 및 구독 정보
   - 주문번호, 판매 ID, 구독 ID, 라이선스 키
   - 상품명, 티어 정보, 결제 상태
   - 환불, 취소, 종료, 분쟁 상태
   - 결제 제공자가 전달한 이메일 주소
   - 결제 제공자가 전달한 이름 정보

운영자는 주민등록번호, 계좌번호, 카드번호, 비밀번호, API 키, 민감한 금융정보의 입력을 요구하지 않습니다.

## 3. 보유 및 이용 기간

운영자는 처리 목적 달성에 필요한 범위에서만 개인정보를 보유하며, 기간이 지나면 삭제합니다.

1. Discord 명령 실행 로그, 오류 로그, 실행 메타데이터
   - 최대 1년
2. 보고서 파일, JSON 결과, 로컬 산출물
   - 최대 1년
3. 문의 또는 분쟁 대응 기록
   - 처리 완료 후 최대 3년
4. 결제 이벤트, 구독 상태 이력, 환불·취소·분쟁 대응 기록
   - 구독 종료 또는 처리 완료 후 최대 5년 또는 관련 법령상 보존 의무 기간

운영자는 보유기간이 지난 결제 이벤트, 구독 이력, 만료된 claim code, 캐시 메타데이터, 보고서 산출물 및 관련 로그를 주기적 배치 작업으로 자동 삭제할 수 있습니다.

## 4. 제3자 제공 및 외부 서비스

운영자는 원칙적으로 개인정보를 제3자에게 임의 제공하지 않습니다. 다만 서비스 운영을 위해 아래 외부 서비스를 이용할 수 있습니다.

1. Discord
   - 명령 수신, 메시지 송수신, 파일 업로드, 사용자 식별
2. OpenAI 또는 관련 모델 제공 인프라
   - 리서치 실행, 결과 생성
3. Gumroad 또는 운영자가 사용하는 결제·구독 제공자
   - 결제 처리, 구독 상태 확인, 환불·취소·분쟁 상태 통지, 영수증 또는 라이선스 검증

법령에 특별한 규정이 있거나 적법한 기관 요청이 있는 경우에는 관련 법령에 따라 정보를 제공할 수 있습니다.

## 5. 국외 처리 가능성

서비스는 Discord, OpenAI, Gumroad 등 해외 인프라를 포함한 외부 서비스를 이용할 수 있습니다. 이에 따라 일부 정보가 대한민국 외 지역의 서버에서 처리되거나 저장될 수 있습니다.

운영자는 서비스 운영에 필요한 범위에서만 이러한 외부 서비스를 이용합니다.

## 6. 파기

운영자는 보유기간 경과 또는 처리 목적 달성 시 지체 없이 개인정보를 삭제합니다.

1. 전자적 파일은 복구가 어렵도록 삭제합니다.
2. 법령상 별도 보존이 필요한 정보는 일반 데이터와 분리하여 보관한 뒤 기간 종료 후 삭제합니다.

## 7. 이용자 권리 안내

운영자는 서비스 운영상 필요한 최소 메타데이터만 제한적으로 보관합니다. 이용자가 프롬프트나 질문에 임의로 입력한 개인정보는 별도 색인 체계로 분류하지 않습니다. 따라서 운영자가 이미 보유한 최소 정보만으로 요청 대상을 식별할 수 있는 경우에 한해 열람, 정정, 삭제, 처리정지 요청을 검토할 수 있습니다.

다음 경우에는 요청 처리가 제한될 수 있습니다.

1. 요청 대상 정보를 운영자가 별도로 식별할 수 없는 경우
2. 요청 처리를 위해 추가 개인정보 제출이 필요한 경우
3. 법령상 제한이 있는 경우
4. 다른 사람의 권리나 서비스 보안을 침해할 우려가 있는 경우

## 8. 안전성 확보 조치

운영자는 합리적인 범위에서 다음과 같은 보호조치를 적용합니다.

1. 접근 권한 최소화
2. 로그 및 산출물 접근 통제
3. 비밀정보 분리 관리
4. 보안 업데이트 및 점검
5. 결제 이벤트 감사 로그와 구독 상태 변경 이력의 제한적 보관
