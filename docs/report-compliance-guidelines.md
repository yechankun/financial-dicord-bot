# Report Compliance Guidelines

이 문서는 Discord 봇이 `context-report-studio`를 호출할 때 추가로 강제할 보고서 작성 규칙이다.

## 목적

- 보고서를 `정보 제공용 리서치`로 유지한다.
- 특정 이용자에 대한 투자 권유나 맞춤형 자문으로 읽힐 표현을 줄인다.
- 대한민국 자본시장법상 오인 가능성이 높은 문구를 피한다.

## 필수 규칙

1. 보고서 첫 페이지 상단 또는 하단에 아래 고지를 넣는다.
   - `본 자료는 정보 제공용 리서치입니다. 투자 권유 또는 투자자문이 아니며, 이용자의 개별 사정을 반영하지 않습니다. 투자 판단과 책임은 이용자 본인에게 있습니다.`
2. 개별 이용자의 보유종목, 손익, 매수가, 목표가, 손절가, 자산규모, 위험성향을 전제로 한 맞춤형 표현을 쓰지 않는다.
3. `지금 사라`, `반드시 팔아라`, `매수 추천`, `수익 보장`, `목표 수익률 달성` 같은 직접 권유형 문구를 쓰지 않는다.
4. 행동 문구는 아래 같은 중립 표현으로 바꾼다.
   - `매수` -> `관심 후보`, `우선 관찰`, `시나리오상 우위`
   - `매도` -> `리스크 확대`, `약세 시나리오`, `모니터링 강화`
   - `비중 확대` -> `상대 우선순위 상향`
   - `헤지` -> `방어 표현 수단`
5. `Capital Allocation` 섹션은 실제 자금 배분 지시가 아니라 `시나리오별 상대 우선순위` 또는 `표현 수단 비교` 중심으로 쓴다.
6. `add / trim / exit / invalidate / pivot`가 필요한 경우에도 실제 주문 지시처럼 쓰지 말고, 아래처럼 조건문으로 일반화한다.
   - `add` -> `추가 확인 신호`
   - `trim` -> `과열 경고 신호`
   - `exit` -> `시나리오 파기 신호`
7. 종목, ETF, 원자재, 금리, 통화 언급은 모두 일반 정보와 공개 데이터 기반 시나리오로 서술하고, 특정 이용자에게 적용되는 최종 행동으로 단정하지 않는다.
8. Page 1 headline과 summary는 강한 결론형 문장을 쓰더라도 `정보 제공용`, `시나리오`, `관찰`, `리스크` 문맥을 유지한다.

## 허용 표현 예시

- `향후 10영업일 동안 상대적으로 강할 가능성이 높은 관심 후보`
- `현재 공개 데이터 기준 우선 관찰 섹터`
- `강세 시나리오가 유지되는 동안 상대 우위가 있는 표현 수단`
- `약세 전환을 의심해야 할 무효화 신호`

## 피해야 할 표현 예시

- `지금 매수해야 한다`
- `이번 주 안에 사라`
- `이 가격에서 반드시 비중을 늘려라`
- `손실 없이 수익 가능하다`
- `회원 전용 종목 추천`
